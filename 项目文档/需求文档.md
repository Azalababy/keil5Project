# 校园外卖系统 - 项目需求文档

## 1. 项目背景与目标

### 1.1 项目背景

随着校园生活节奏的加快，大学生对便捷外卖服务的需求日益增长。传统的电话订餐或第三方平台存在配送慢、价格高、服务质量参差不齐等问题。为此，我们开发了一套专门面向校园场景的校园外卖系统，旨在为大学生提供快速、便捷、优惠的外卖订餐服务。

### 1.2 项目目标

- **快速订餐**: 用户能够在最短时间内完成订餐流程
- **即时配送**: 骑手能够快速接单并完成配送
- **商家管理**: 为商家提供便捷的订单管理和商品管理功能
- **数据分析**: 为管理员提供全面的数据统计和决策支持
- **优惠券系统**: 通过优惠券提升用户活跃度和消费频次

## 2. 系统用户角色

系统包含四种主要角色，每种角色有独立的操作界面和功能权限：

| 角色 | 说明 | 主要功能 |
|------|------|----------|
| 用户 | 普通消费者 | 浏览商家、下单订餐、领取优惠券、评价订单 |
| 商家 | 餐饮商家 | 管理商品、接收订单、制作餐品、查看销售统计 |
| 骑手 | 配送人员 | 接单取餐、配送订单、查看收入统计 |
| 管理员 | 系统维护 | 用户管理、商家审核、数据统计、系统配置 |

## 3. 功能需求

### 3.1 用户端功能

#### 3.1.1 用户注册与登录

- **注册方式**: 手机号注册，设置密码
- **登录方式**: 用户名+密码登录
- **角色选择**: 用户、商家、骑手三种角色统一登录入口
- **状态管理**: 登录状态持久化，刷新页面保持登录

#### 3.1.2 首页功能

- **横幅区域**: 
  - "立即点餐"按钮 - 跳转至商家列表
  - "了解更多"按钮 - 展示平台介绍
  - "每日抢券"按钮 - 跳转至优惠券页面
- **商家列表**: 展示所有营业中的商家
- **分类展示**: 按商家类型或菜品种类分类展示

#### 3.1.3 商家详情页

- **商家信息**: 展示商家名称、logo、评分、起送价、配送费等信息
- **菜品列表**: 分类展示商家菜品，包含名称、价格、图片、描述
- **购物车**: 添加菜品至购物车，支持数量调整
- **下单功能**: 
  - 选择收货地址（必填）
  - 填写联系电话（必填）
  - 选择优惠券（可选）
  - 备注信息
  - 显示原价和实付金额

#### 3.1.4 优惠券系统

- **优惠券领取**:
  - "每日抢券"入口
  - 优惠券面额: 2元、5元、10元、20元
  - 使用门槛: 部分优惠券需满足最低消费金额，部分无门槛
  - 过期日期: 每张优惠券有明确的过期时间
  - 领取限制: 每种优惠券每人限领一张
  - 库存显示: 实时显示剩余数量

- **优惠券使用**:
  - 下单时自动显示可用优惠券
  - 系统自动过滤不满足条件的优惠券
  - 使用后订单金额自动扣除优惠金额
  - 记录优惠券使用状态（未使用、已使用、已过期）

#### 3.1.5 订单管理

- **订单列表**: 展示用户的所有订单
- **订单状态**: 
  - 待支付（用户下单后等待付款）
  - 已支付（等待商家接单）
  - 商家已接单（等待商家制作完成）
  - 制作中（商家正在准备餐品）
  - 待取餐（餐品已准备好，等待骑手取餐）
  - 配送中（骑手正在配送）
  - 已完成（订单已完成）
  - 已取消（订单被取消）
- **订单详情**: 查看订单的完整信息，包括菜品、地址、骑手信息等
- **订单评价**: 对已完成订单进行评价，包含评分和评价内容

### 3.2 商家端功能

#### 3.2.1 商家后台首页

- **今日数据概览**:
  - 今日订单数
  - 今日收入
  - 今日浏览量
  - 待处理订单数
- **订单趋势图**: 展示近7天的订单数量变化
- **收入统计**: 展示收入金额的柱状图

#### 3.2.2 商家登录与注册

- **注册信息**: 商家名称、联系电话、地址、logo
- **登录凭证**: 用户名+密码
- **状态管理**: 登录状态持久化

#### 3.2.3 商家管理

- **商家信息编辑**: 修改商家名称、logo、营业时间等
- **营业状态**: 开启/关闭营业

#### 3.2.4 菜品管理

- **菜品列表**: 展示所有菜品
- **添加菜品**: 
  - 菜品名称（必填）
  - 价格（必填）
  - 分类（必填）
  - 图片（必填）
  - 描述（选填）
  - 库存数量（必填）
- **编辑菜品**: 修改菜品信息
- **删除菜品**: 删除菜品
- **上下架管理**: 控制菜品是否对外展示

#### 3.2.5 订单管理

- **待处理订单**: 商家接单或拒单
- **处理流程**:
  - 商家可选择接受或拒绝订单
  - 拒单需输入原因
  - 接受后进入制作状态
  - 制作完成后标记"待取餐"
- **订单筛选**: 按状态筛选订单
- **订单详情**: 查看订单的完整信息

#### 3.2.6 评价管理

- **评价列表**: 查看用户对本店订单的评价
- **评价内容**: 包含评分和评价内容
- **评价时间**: 显示评价时间

### 3.3 骑手端功能

#### 3.3.1 骑手登录

- **登录凭证**: 用户名+密码
- **状态管理**: 登录状态持久化

#### 3.3.2 骑手信息

- **个人信息**: 显示骑手姓名、手机号
- **在线状态**: 上线/下线切换
- **统计数据**: 
  - 今日订单数
  - 今日收入
  - 平均配送时长
  - 配送完成率

#### 3.3.3 订单管理

- **待接订单**: 展示等待骑手接单的配送任务
  - 显示配送费（按订单金额的20%计算）
  - 显示取货地址和送货地址
  - 显示商品列表和总金额
  - 接单和拒单功能

- **当前配送**:
  - 展示正在配送的订单
  - 配送时间计时器
  - 状态更新: 已取货 → 配送中 → 已送达
  - 支持同时接多单配送

- **历史订单**: 展示已完成和已取消的订单
  - 显示配送时长
  - 显示配送费

#### 3.3.4 拒单功能

- **二次确认**: 需输入"确认拒单"进行二次确认
- **拒单原因**: 可选填拒单原因
- **拒单限制**: 拒单后订单需重新分配

#### 3.3.5 用户评价

- **评价列表**: 展示用户对骑手的评价
- **评价内容**: 包含星级评分和评价内容

### 3.4 管理端功能

#### 3.4.1 管理后台首页

- **数据概览**:
  - 总用户数
  - 总商家数
  - 总订单数
  - 总收入
- **订单趋势**: 展示订单数量变化折线图
- **收入统计**: 展示收入金额柱状图

#### 3.4.2 用户管理

- **用户列表**: 展示所有注册用户
- **用户详情**: 查看用户信息
- **用户禁用**: 禁用违规用户

#### 3.4.3 商家管理

- **商家列表**: 展示所有商家
- **商家审核**: 审核新注册的商家
- **商家详情**: 查看商家信息、菜品、订单等
- **商家禁用**: 禁用违规商家

#### 3.4.4 订单管理

- **订单列表**: 展示所有订单
- **订单详情**: 查看订单完整信息
- **订单统计**: 按日期、商家、状态等维度统计

#### 3.4.5 骑手管理

- **骑手列表**: 展示所有骑手
- **骑手详情**: 查看骑手信息、接单统计、收入统计
- **骑手禁用**: 禁用违规骑手

#### 3.4.6 评价管理

- **评价列表**: 展示所有用户评价
- **评价统计**: 统计评价的平均分、好评率等

## 4. 非功能需求

### 4.1 性能需求

- **响应时间**: 页面加载时间不超过3秒
- **并发处理**: 支持至少100个并发用户
- **数据处理**: 单次查询响应时间不超过500ms

### 4.2 安全性需求

- **用户认证**: 所有操作需通过登录认证
- **密码加密**: 用户密码使用加密存储
- **输入验证**: 所有用户输入进行合法性验证
- **SQL注入防护**: 使用参数化查询防止SQL注入

### 4.3 可用性需求

- **系统可用性**: 7x24小时可用
- **故障恢复**: 系统故障后能在30分钟内恢复
- **数据备份**: 每日自动备份数据库

### 4.4 兼容性需求

- **浏览器兼容**: 支持Chrome、Firefox、Safari、Edge等主流浏览器
- **移动端适配**: 支持iOS和Android系统的移动设备
- **响应式设计**: 页面能够适应不同屏幕尺寸

### 4.5 数据需求

- **数据完整性**: 确保数据的完整性和一致性
- **数据持久化**: 所有业务数据持久化存储
- **数据迁移**: 支持数据的导入导出

## 5. 业务流程

### 5.1 用户下单流程

```
用户浏览商家 → 选择菜品加入购物车 → 确认订单信息 → 选择收货地址 → 选择优惠券 → 提交订单 → 支付成功 → 等待商家接单 → 商家制作 → 骑手接单取餐 → 骑手配送 → 确认送达 → 订单完成
```

### 5.2 商家接单流程

```
接收订单通知 → 确认订单信息 → 商家接单/拒单 → 接受则开始制作 → 制作完成标记待取餐 → 等待骑手取餐
```

### 5.3 骑手配送流程

```
查看待接订单 → 选择订单接单 → 前往商家取餐 → 确认取货 → 开始配送 → 到达目的地 → 确认送达 → 完成配送
```

### 5.4 优惠券使用流程

```
用户进入抢券页面 → 浏览可用优惠券 → 点击领取 → 优惠券进入我的卡包 → 下单时选择可用优惠券 → 系统自动计算优惠金额 → 完成支付
```

## 6. 数据字典

### 6.1 用户表 (users)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键ID |
| username | VARCHAR(50) | 用户名 |
| password | VARCHAR(100) | 密码（加密存储） |
| phone | VARCHAR(20) | 手机号 |
| created_at | DATETIME | 创建时间 |

### 6.2 商家表 (shops)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键ID |
| user_id | BIGINT | 关联用户ID |
| name | VARCHAR(100) | 商家名称 |
| logo | VARCHAR(200) | 商家logo |
| phone | VARCHAR(20) | 联系电话 |
| address | VARCHAR(200) | 商家地址 |
| description | VARCHAR(500) | 商家描述 |
| status | ENUM | 营业状态 |

### 6.3 菜品表 (dishes)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键ID |
| shop_id | BIGINT | 关联商家ID |
| name | VARCHAR(100) | 菜品名称 |
| price | DECIMAL(10,2) | 价格 |
| image | VARCHAR(200) | 图片 |
| description | VARCHAR(500) | 描述 |
| category | VARCHAR(50) | 分类 |
| stock | INT | 库存 |
| status | ENUM | 上下架状态 |

### 6.4 订单表 (orders)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键ID |
| user_id | BIGINT | 用户ID |
| shop_id | BIGINT | 商家ID |
| rider_id | BIGINT | 骑手ID |
| address | VARCHAR(200) | 收货地址 |
| phone | VARCHAR(20) | 联系电话 |
| total_price | DECIMAL(10,2) | 订单总金额 |
| actual_price | DECIMAL(10,2) | 实付金额 |
| coupon_id | BIGINT | 使用优惠券ID |
| status | ENUM | 订单状态 |
| created_at | DATETIME | 创建时间 |

### 6.5 优惠券表 (coupons)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键ID |

| expire_date | DATE | 过期日期 |
| description | VARCHAR(200) | 描述 |
| total_quantity | INT | 总数量 |
| remaining_quantity | INT | 剩余数量 |
| status | ENUM | 状态 |

### 6.6 用户优惠券表 (user_coupons)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键ID |
| user_id | BIGINT | 用户ID |
| coupon_id | BIGINT | 优惠券ID |
| status | ENUM | 状态（未使用/已使用/已过期） |
| order_id | BIGINT | 使用订单ID |

### 6.7 骑手表 (riders)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键ID |
| user_id | BIGINT | 关联用户ID |
| name | VARCHAR(50) | 姓名 |
| phone | VARCHAR(20) | 手机号 |
| status | ENUM | 状态 |
