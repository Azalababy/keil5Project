## 1. 项目架构概览

```
STM32F103C8 (72MHz Cortex-M3)
    │
    ├── USART1 (PA9/PA10) ──── 串口调试输出 (9600bps)
    │      └── Serial_Printf(1, ...)  → PC端串口助手查看
    │
    ├── USART2 (PA2/PA3)  ──── ESP8266 WiFi模块 (115200bps)
    │      └── AT指令交互 + TCP数据收发
    │
    ├── PA8 ──────────────── LED (推挽输出, Bit_SET=亮)
    │
    └── I2C OLED ──────────── 显示IP地址等调试信息
```

**文件分工：**

| 文件 | 职责 |
|------|------|
| `main.c` | 系统初始化、主循环调度 |
| `Serial.c` | 串口1/2的初始化、发送、接收中断 |
| `esp8266.c` | ESP8266初始化(WiFi连接+TCP Server) + HTTP请求处理 |

### main() 函数

```c
int main(void)
{
    RCC_Configuration();    // ① 配置系统时钟: HSE(8MHz) → PLL×9 = 72MHz
    Delay_Init();           // ② 初始化SysTick定时器，产生1ms时基
    LED_Init();             // ③ 初始化PA8为推挽输出，默认低电平(LED灭)
    OLED_Init();            // ④ 初始化I2C OLED显示屏
    OLED_Clear();           // ⑤ 清屏

    ESP8266_Init();         // ⑥ ESP8266完整初始化流程 (详见第4节)

    while (1) {             // ⑦ 主循环
        ESP8266_HandleClient();  // 每循环检查是否有HTTP客户端请求
        Delay_ms(10);            // 休眠10ms，避免过于频繁轮询
    }
}
```

**执行流程：** 系统启动 → 时钟72MHz → 各外设初始化 → ESP8266连WiFi+开TCP Server → 死循环等待HTTP请求

### RCC_Configuration() 函数 — 时钟树配置

```c
void RCC_Configuration(void)
{
    ErrorStatus HSEStartUpStatus;
    RCC_DeInit();                       // 复位RCC寄存器到默认值

    RCC_HSEConfig(RCC_HSE_ON);          // 开启外部高速晶振 HSE (8MHz)
    HSEStartUpStatus = RCC_WaitForHSEStartUp();  // 等待HSE稳定启动
    if (HSEStartUpStatus == SUCCESS) {
        // ====== 配置Flash等待周期 ======
        FLASH_PrefetchBufferCmd(FLASH_PrefetchBuffer_Enable);  // 使能预取缓冲，提升取指速度
        FLASH_SetLatency(FLASH_Latency_2);  // 72MHz需要2个Flash等待周期

        // ====== 配置AHB/APB总线分频 ======
        RCC_HCLKConfig(RCC_SYSCLK_Div1);    // AHB时钟 = 系统时钟 / 1 = 72MHz
        RCC_PCLK1Config(RCC_HCLK_Div2);     // APB1时钟 = 72MHz / 2 = 36MHz (USART2在此总线)
        RCC_PCLK2Config(RCC_HCLK_Div1);     // APB2时钟 = 72MHz / 1 = 72MHz (GPIO/USART1在此总线)

        // ====== 配置PLL锁相环 ======
        RCC_PLLConfig(RCC_PLLSource_HSE_Div1, RCC_PLLMul_9);
        // HSE 8MHz / 1 × 9 = 72MHz 系统时钟
        RCC_PLLCmd(ENABLE);                 // 开启PLL
        while (RCC_GetFlagStatus(RCC_FLAG_PLLRDY) == RESET);  // 等待PLL锁定
        RCC_SYSCLKConfig(RCC_SYSCLKSource_PLLCLK);            // 切换系统时钟源为PLL
        while (RCC_GetSYSCLKSource() != 0x08);  // 确认已切换到PLL (0x08 = PLL)
    }
    // ====== 外设时钟使能 ======
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA | RCC_APB2Periph_GPIOB | RCC_APB2Periph_AFIO, ENABLE);
    // GPIOA(USART1/2,LED,OLED) + GPIOB + 复用时钟
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_USART2, ENABLE);
    // USART2在APB1总线上 (注意: USART1在APB2)
}
```

**时钟树总结：**
```
HSE 8MHz ──→ PLL (×9) ──→ SYSCLK 72MHz
                              ├── HCLK (AHB)  72MHz
                              │     └── APB1  36MHz → USART2
                              └── APB2        72MHz → GPIOA/B, USART1
```

### Serial_Init() — 串口初始化

```c
void Serial_Init(uint8_t USART_Name)
```

根据 `USART_Name` 参数初始化USART1或USART2。

#### USART1 (调试串口)
```c
if (USART_Name == 1)
{
    // ① 使能时钟
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_USART1, ENABLE);  // USART1在APB2总线
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);   // GPIOA在APB2

    // ② 配置TX引脚 PA9 — 复用推挽输出
    GPIO_InitTypeDef GPIO_InitStructure;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_PP;     // 复用推挽输出 (USART TX)
    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_9;           // PA9 = USART1_TX
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;   // 50MHz驱动能力
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    // ③ 配置RX引脚 PA10 — 上拉输入
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPU;       // 上拉输入 (USART RX)
    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_10;          // PA10 = USART1_RX
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    // ④ 配置USART参数
    USART_InitTypeDef USART_InitStructure;
    USART_InitStructure.USART_BaudRate = 9600;          // 波特率 9600 (调试用)
    USART_InitStructure.USART_HardwareFlowControl = USART_HardwareFlowControl_None; // 无硬件流控
    USART_InitStructure.USART_Mode = USART_Mode_Tx | USART_Mode_Rx;    // 收发双工
    USART_InitStructure.USART_Parity = USART_Parity_No;  // 无校验
    USART_InitStructure.USART_StopBits = USART_StopBits_1;  // 1停止位
    USART_InitStructure.USART_WordLength = USART_WordLength_8b;  // 8数据位
    // 总结: 9600-8-N-1
    USART_Init(USART1, &USART_InitStructure);

    // ⑤ 使能接收中断
    USART_ITConfig(USART1, USART_IT_RXNE, ENABLE);
    // RXNE = RX Not Empty — 每收到一个字节就触发中断

    // ⑥ 配置NVIC中断优先级
    NVIC_PriorityGroupConfig(NVIC_PriorityGroup_2);      // 2位抢占 + 2位子优先级
    NVIC_InitTypeDef NVIC_InitStructure;
    NVIC_InitStructure.NVIC_IRQChannel = USART1_IRQn;    // USART1中断通道
    NVIC_InitStructure.NVIC_IRQChannelCmd = ENABLE;
    NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 1;  // 抢占优先级1
    NVIC_InitStructure.NVIC_IRQChannelSubPriority = 1;        // 子优先级1
    NVIC_Init(&NVIC_InitStructure);

    // ⑦ 使能USART1
    USART_Cmd(USART1, ENABLE);
}
```

#### USART2 (ESP8266通信串口)
与USART1基本相同，差异点：
| 参数 | USART1 (调试) | USART2 (ESP8266) |
|------|:---:|:---:|
| 总线 | APB2 | **APB1** |
| TX引脚 | PA9 | **PA2** |
| RX引脚 | PA10 | **PA3** |
| 波特率 | 9600 | **115200** |
| 时钟使能 | `RCC_APB2Periph_USART1` | `RCC_APB1Periph_USART2` |
| 中断号 | `USART1_IRQn` | `USART2_IRQn` |

### Serial_SendByte() — 发送一个字节
```c
void Serial_SendByte(uint8_t USART_Name, uint8_t Byte)
{
    if (USART_Name == 1)
    {
        USART_SendData(USART1, Byte);               // 写入数据寄存器
        while (USART_GetFlagStatus(USART1, USART_FLAG_TXE) == RESET);
        // TXE = Transmit Empty — 等待硬件把数据发完才能写下一个字节
    }
    else if (USART_Name == 2)
    {
        USART_SendData(USART2, Byte);
        while (USART_GetFlagStatus(USART2, USART_FLAG_TXE) == RESET);
    }
}
```
> **TXE vs TC:** TXE在数据从寄存器移到移位器后置位（可写下一字节）；TC在整个帧发完后置位。这里用TXE效率更高。

### Serial_Pow() — 幂运算辅助函数
```c
uint32_t Serial_Pow(uint32_t x, uint32_t y)
{
    uint32_t Result = 1;
    while (y--) { Result *= x; }  // 计算 x^y
    return Result;
}
```

### Serial_SendNumber() — 发送数字
```c
void Serial_SendNumber(uint8_t USART_Name, uint32_t Number, uint16_t Length)
{
    for (uint16_t i = 0; i < Length; i++)
    {
        // 取出第(Length-i-1)位的数字，转成ASCII
        Serial_SendByte(USART_Name, Number / Serial_Pow(10, Length - i - 1) % 10 + '0');
    }
}
// 示例: SendNumber(1, 123, 5) → 发送 "00123"
```

### Serial_Printf() — 格式化输出（可变参数）
```c
void Serial_Printf(uint8_t USART_Name, char *format, ...)
{
    char String[100];           // 格式化缓冲区 (最多100字节)
    va_list arg;                 // 可变参数列表
    va_start(arg, format);       // 初始化arg，指向format后的第一个参数
    vsprintf(String, format, arg);  // 格式化写入String
    va_end(arg);                 // 释放arg
    Serial_SendString(USART_Name, String);  // 发送格式化后的字符串
}
// 示例: Serial_Printf(1, "IP=%s\r\n", ip) → USART1输出 "IP=10.126.41.134\r\n"
```

### 中断服务函数

#### USART1_IRQHandler() — USART1接收中断
```c
void USART1_IRQHandler(void)
{
    if (USART_GetITStatus(USART1, USART_IT_RXNE) != RESET)  // 确认是RXNE中断
    {
        uint8_t RxData = USART_ReceiveData(USART1);         // 读出接收到的字节
        USART1_RX_BUF[USART1_RX_LEN++] = RxData;            // 存入缓冲区，索引+1
        if (USART1_RX_LEN >= 1023) USART1_RX_LEN = 0;      // 防止溢出，回绕
        USART_ClearITPendingBit(USART1, USART_IT_RXNE);     // 清除中断标志
    }
}
```

> **为什么需要读 DR 寄存器？** 读 `USART_ReceiveData()` 会自动清除RXNE标志。如果收到数据不读，RXNE一直置位，会反复触发中断导致死循环。

#### USART2_IRQHandler() — USART2接收中断 (ESP8266数据)
逻辑与USART1完全一致，操作USART2的缓冲区和寄存器。

### ClearRxBuf() — 清空接收缓冲区（static，仅本文件可见）
```c
static void ClearRxBuf(void)
{
    USART2_RX_LEN = 0;              // 重置数据长度（逻辑清空）
    memset(USART2_RX_BUF, 0, 1024); // 用0填充全部1024字节（物理清空）
```

### ESP8266_Init() — 完整初始化流程

```
时序:
  退出透传 → AT测试 → 设STA模式 → 连WiFi → 获取IP → 开多连接 → 开TCP Server(80)
```

#### 第一步：初始化串口 + 退出透传
```c
Serial_Init(1);           // 初始化USART1 (调试输出)
Serial_Init(2);           // 初始化USART2 (ESP8266通信, 115200bps)
Delay_ms(2000);           // 等待ESP8266上电稳定 (2秒)

ClearRxBuf();
Serial_SendString(2, "+++");     // 发送 "+++" 退出透传模式
Delay_ms(1500);                  // 等待ESP8266响应 (透传退出慢)
ClearRxBuf();
```
> **透传模式：** ESP8266在此模式下把所有串口数据直接发到TCP另一端，不解析AT指令。`+++` (不带\r\n) 可退出透传回到AT模式。

#### 第二步：AT通信测试
```c
Serial_SendString(2, "AT\r\n");   // 发送AT测试指令
Delay_ms(800);                     // 等待响应
if (strstr(USART2_RX_BUF, "OK") == NULL) {   // 在返回数据中搜索"OK"
    Serial_SendString(1, "AT fail\r\n");      // 没找到 → 通信失败
    return;                                    // 直接返回，不再继续
}
Serial_SendString(1, "AT OK\r\n");  // 通信正常 → 调试串口输出
```
> ESP8266 成功响应 `AT\r\n` → 返回 `AT\r\n\r\nOK\r\n`。strstr 搜索 "OK" 来判断成功。

#### 第三步：设置WiFi模式
```c
Serial_SendString(2, "AT+CWMODE=1\r\n");  // 设为Station模式 (客户端)
Delay_ms(800);
ClearRxBuf();
```
> `AT+CWMODE=1` → 设置为 **Station模式（客户端）**，ESP8266会连接到一个已有的WiFi热点。

#### 第四步：连接WiFi
```c
Serial_SendString(1, "WiFi...\r\n");                      // 调试提示
ClearRxBuf();
Serial_SendString(2, "AT+CWJAP=\"a\",\"12345678\"\r\n");  // 连接WiFi: SSID="a", 密码="12345678"

uint8_t ok = 0;
uint32_t elapsed = 0;
while (elapsed < 15000) {                                 // 最多等15秒
    if (strstr(USART2_RX_BUF, "WIFI GOT IP") != NULL) { ok = 1; break; }  // 获取到IP → 成功
    if (strstr(USART2_RX_BUF, "FAIL") != NULL) break;     // 返回FAIL → 失败
    Delay_ms(20); elapsed += 20;                           // 每20ms检查一次
}
if (!ok) { Serial_SendString(1, "WiFi fail\r\n"); return; }
Serial_SendString(1, "WiFi OK\r\n");
```
> `AT+CWJAP="ssid","password"` — 连接到指定WiFi接入点。连接成功后ESP8266会异步返回 `WIFI CONNECTED` 和 `WIFI GOT IP`。

#### 第五步：获取IP地址
```c
char ip[16] = {0};                        // IP字符串缓冲区 (最多15字符+'\0')
for (int r = 0; r < 5; r++) {            // 最多重试5次
    ClearRxBuf();
    Serial_SendString(2, "AT+CIFSR\r\n"); // 查询本机IP
    Delay_ms(1500);
    char *p = strstr(USART2_RX_BUF, "STAIP,\"");          // 搜索 "STAIP,""
    if (!p) p = strstr(USART2_RX_BUF, "+CIFSR:STAIP,\""); // 兼容不同固件格式
    if (p) {
        while (*p && *p != '\"') p++;      // 跳到第一个引号后
        if (*p == '\"') p++;
        int i = 0;
        while (*p && *p != '\"' && i < 15) ip[i++] = *p++;  // 复制IP字符，直到第二个引号
        ip[i] = '\0';                      // 字符串结束符
        break;
    }
}
```
> ESP8266 返回示例: `+CIFSR:STAIP,"10.126.41.134"`。解析逻辑：找双引号之间的内容。

#### 第六步：OLED显示IP
```c
OLED_Clear();
if (ip[0]) {
    OLED_ShowString(1, 1, "IP:");          // 第1行显示 "IP:"
    OLED_ShowString(2, 1, ip);             // 第2行显示实际IP
    Serial_Printf(1, "IP=%s\r\n", ip);     // 同时发送到调试串口
} else {
    OLED_ShowString(1, 1, "No IP");        // 获取失败
}
```

#### 第七步：开启TCP Server
```c
Serial_SendString(2, "AT+CIPMUX=1\r\n");    // ① 使能多连接模式 (允许多个客户端同时连接)
Delay_ms(800);
ClearRxBuf();

Serial_SendString(2, "AT+CIPSERVER=0\r\n"); // ② 先关闭之前的Server (如果有的话，避免冲突)
Delay_ms(800);
ClearRxBuf();

Serial_SendString(2, "AT+CIPSERVER=1,80\r\n"); // ③ 开启TCP Server，监听80端口(HTTP默认端口)
Delay_ms(1500);
ClearRxBuf();

Serial_SendString(1, "READY\r\n");           // 初始化完成
```
> **CIPMUX:** `0`=单连接 (只能连一个客户端), `1`=多连接 (可连多个，每个有link_id)。作为Server必须用多连接。

> **CIPSERVER:** `0`=关闭Server, `1,80`=开启Server端口80。

### ESP8266_HandleClient() — HTTP客户端请求处理

**调用时机：** 主循环中每10ms调用一次。

```c
void ESP8266_HandleClient(void)
{
    // ===== ① 检查是否有新请求 =====
    char *ipd = strstr(USART2_RX_BUF, "+IPD");
    if (ipd == NULL) return;         // 没有 "+IPD" → 没有客户端请求，直接返回
```
> ESP8266收到客户端数据时，通过串口输出格式: `+IPD,<link_id>,<data_len>:<data>`
> 示例: `+IPD,0,386:GET /ON HTTP/1.1\r\nHost: ...`

```c
    // ===== ② 解析link_id和数据长度 =====
    int link_id = 0, data_len = 0;
    if (sscanf(ipd, "+IPD,%d,%d:", &link_id, &data_len) != 2) {
        ClearRxBuf();       // 解析失败，格式异常
        return;
    }
    // sscanf 从 "+IPD,0,386:..." 中提取 link_id=0, data_len=386
```

```c
    // ===== ③ 定位HTTP请求内容 =====
    char *req = strstr(ipd, ":");
    if (req == NULL) { ClearRxBuf(); return; }
    req++;  // 跳过冒号，req现在指向HTTP请求的第一个字节
```

```c
    // ===== ④ 解析HTTP请求路径，控制LED =====
    int st;
    if      (strstr(req, "GET /ON "))  { LED_On();  st = 1; }   // 请求 /ON → 开灯
    else if (strstr(req, "GET /OFF ")) { LED_Off(); st = 0; }   // 请求 /OFF → 关灯
    else {
        // 非ON/OFF请求（如首次访问/）→ 不改变LED状态，仅读取
        st = (GPIO_ReadOutputDataBit(GPIOA, GPIO_Pin_8) == Bit_SET) ? 1 : 0;
    }
```
> `GPIO_ReadOutputDataBit(GPIOA, GPIO_Pin_8)` — 读取PA8的当前输出电平。Bit_SET=高电平(LED亮)，RESET=低电平(LED灭)。

#### ⑤ 构造HTTP响应HTML

```c
    const char *fmt = ...;  // HTML模板字符串（含CSS样式），此处省略
```
> **%% 的作用：** CSS中的 `50%` 在snprintf的format字符串中必须写成 `50%%`，因为 `%` 是格式占位符前缀。`%%` 转义输出一个字面的 `%`。
> 
> **%s 占位符：** 第1个 `%s` → LED圆点样式("on"/"off")，第2个 `%s` → LED状态文字("ON"/"OFF")

HTML结构：
```
┌─────────────────────────────────┐
│         LED Control             │  ← h1 标题
│      STM32 + ESP8266            │  ← sub 副标题
│         ┌───┐                   │
│         │ ● │                   │  ← LED指示圆 (on橙金/off灰)
│         └───┘                   │
│       PA8: ON                   │  ← 状态文字
│    [  ON  ]  [  OFF  ]          │  ← 按钮 (暖橙/灰渐变)
└─────────────────────────────────┘
```

#### ⑥ 格式化HTML
```c
    static char html[2000];      // static! 不占栈空间 (栈仅1KB)
    int n = snprintf(html, sizeof(html), fmt, st ? "on" : "off", st ? "ON" : "OFF");
    // snprintf 返回完整HTML长度（即使超过缓冲区也会返回应有的长度）
    // 这里缓冲区2000字节远超实际HTML(~800字节)，所以n=实际长度
```

> **为什么用 static？** STM32F103默认栈空间仅 `0x400 = 1024字节`。如果 `char html[2000]` 是普通局部变量（分配在栈上），会直接栈溢出导致程序崩溃。`static` 将数组移到静态内存区(BSS段)，不占栈。

#### ⑦ 通过ESP8266发送HTTP响应
```c
    char cmd[40];
    snprintf(cmd, sizeof(cmd), "AT+CIPSEND=%d,%d\r\n", link_id, n);
    Serial_SendString(2, cmd);       // 告诉ESP8266: 准备向link_id发送n字节
    Delay_ms(400);                    // 等待ESP8266返回 ">" 提示符

    Serial_SendString(2, html);      // 发送HTML内容
    Delay_ms(400);                    // 等待ESP8266发送完成并返回 "SEND OK"

    snprintf(cmd, sizeof(cmd), "AT+CIPCLOSE=%d\r\n", link_id);
    Serial_SendString(2, cmd);       // 关闭该连接
    Delay_ms(200);

    ClearRxBuf();                     // 清空缓冲区，准备处理下一个请求
}
```

**HTTP请求处理完整流程：**
```
浏览器 → ESP8266 → USART2 → STM32解析
                                 │
                    GET /ON  → LED_On()
                    GET /OFF → LED_Off()
                    GET /    → 读取当前状态
                                 │
STM32 → USART2 → ESP8266 → 浏览器（渲染HTML页面）
```

## 5. ESP8266 四种工作模式

ESP8266 通过 `AT+CWMODE=<mode>` 命令设置工作模式。

| mode | 名称 | 说明 | 典型场景 |
|:---:|------|------|------|
| **1** | **Station (STA)** | 客户端模式，连接到一个已有的WiFi热点 | 本项目使用此模式 |
| **2** | **SoftAP (AP)** | 热点模式，ESP8266自己作为WiFi热点 | ESP8266发WiFi，手机直连 |
| **3** | **Station + SoftAP** | 同时作为客户端和热点 | 既能上网又能本地配置 |

---

### 模式 1 — Station 模式 （客户端）

```
      ┌─────────┐           ┌─────────┐
      │ 路由器   │  ◄──────► │ ESP8266 │── UART ── STM32
      │ (WiFi)  │    WiFi    └─────────┘
      └─────────┘
```

- ESP8266 **连接**到一个已有的WiFi AP（如路由器、手机热点）
- 路由器分配IP给ESP8266
- **本项目：** STM32通过 `AT+CWJAP="a","12345678"` 连接手机热点
- 优点：ESP8266在局域网内，同一WiFi下的设备都可以访问

**AT指令：**
```
AT+CWMODE=1                     ← 设为Station模式
AT+CWJAP="SSID","密码"           ← 连接WiFi
AT+CIFSR                        ← 查询获取到的IP
```

---

### 模式 2 — SoftAP 模式 （热点）

```
                ┌─────────┐
     手机/电脑   │ ESP8266 │── UART ── STM32
     ◄────────► │ (热点)  │
       WiFi     └─────────┘
```

- ESP8266 **自己**建立一个WiFi热点
- 手机/电脑直接连接ESP8266的热点
- ESP8266默认IP: `192.168.4.1`
- 优点：不需要外部路由器，适合户外或现场调试

**AT指令：**
```
AT+CWMODE=2                     ← 设为AP模式
AT+CWSAP="ESP8266","12345678",1,3  ← 设置热点: SSID/密码/通道/加密方式
AT+CIPMUX=1                     ← 多连接模式
AT+CIPSERVER=1,80               ← 开启TCP Server
```

---

### 模式 3 — Station + SoftAP 模式 （混合）

```
      ┌─────────┐           ┌─────────┐
      │ 路由器   │  ◄──────► │         │── UART ── STM32
      └─────────┘    WiFi   │ ESP8266 │
                            │         │
             手机  ◄──────► │         │
                   WiFi    └─────────┘
```

- ESP8266 **同时**连接外部WiFi和自建热点
- 既可以上网（通过路由器），又允许手机直连配置
- 适合需要远程控制 + 本地配置的场景
- **注意：** 实际使用中，两种模式同时工作可能导致性能下降

**AT指令：**
```
AT+CWMODE=3                     ← 设为混合模式
AT+CWJAP="外网SSID","密码"       ← 连接外部WiFi（上网用）
AT+CWSAP="ESP8266","12345678",1,3  ← 同时开启本地热点
```

---

### ⚠️ 关于模式 0

ESP8266 还有 **mode=0**（无WiFi模式），仅作为串口透传设备使用，不常用。

---

### 本项目为什么选 Station 模式？

| 需求 | 方案 |
|------|------|
| 手机/电脑控制LED | 通过浏览器访问ESP8266的Web页面 |
| 局域网内访问 | Station模式连路由器，IP固定在同一网段 |
| 简单可靠 | Station模式比AP模式连接质量更好 |

如果改用 **SoftAP 模式**，需要：手机连接ESP8266的热点 → 浏览器访问 `192.168.4.1`，不需要外部路由器。

## 6. 关键调试信息对照

### ESP8266_Init() 期望的串口输出

| 阶段 | USART1调试输出 | USART2 ESP8266返回 | 说明 |
|------|:---|------|------|
| AT测试 | `AT OK` | `AT\r\n\r\nOK\r\n` | 通信正常 |
| 连接WiFi | `WiFi...` | `WIFI CONNECTED\r\nWIFI GOT IP\r\n` | 连接成功 |
| 连接成功 | `WiFi OK` | — | — |
| 获取IP | `IP=10.126.41.134` | `+CIFSR:STAIP,"10.126.41.134"` | 显示在OLED上 |
| Server启动 | `READY` | `OK` | 准备接收HTTP请求 |

### ESP8266_HandleClient() 处理流程

```
USART2收到: +IPD,0,386:GET /ON HTTP/1.1...
                │
                ▼
          解析 link_id=0, len=386
                │
                ▼
          req = "GET /ON HTTP/1.1..."
                │
                ▼
          strstr(req, "GET /ON ") → 匹配 → LED_On()
                │
                ▼
          构造HTML (snprintf)
                │
                ▼
          AT+CIPSEND=0,<len> → 发送HTML → AT+CIPCLOSE=0
```

### 常见故障排查

| 故障现象 | 可能原因 |
|------|------|
| `AT fail` | ESP8266未上电、TX/RX接反、波特率不匹配(必须115200) |
| `WiFi fail` | WiFi热点未开启、SSID/密码错误、信号太弱 |
| `No IP` | DHCP获取超时、路由器未开启DHCP |
| 网页打不开 | IP地址不对、不在同一局域网、浏览器缓存 |
| 网页无内容 | 栈溢出、HTML被截断、`snprintf` 长度计算错误 |
| 网页响应慢 | Delay_ms设置过长、WiFi信号差 |
